## Evidence Summary
- **Total signals:** 2
- **Date range:** April - June 2025 (2 months, recent)
- **Community:** Reddit r/podcasting (same cohort, cross-validated)
- **Tool:** Descript (podcast/video editing software)
- **Core problems:**
  1. Frequent updates breaking workflows
  2. Old projects breaking after updates
  3. Poor/no communication about changes
  4. Learning curve for every update
  5. Extra time needed to accommodate changes
  6. Fear and anxiety around updates

## Emotional Signals
- "Crushing me"
- "Exhausted"
- "A little scared everytime I see an update"
- "It was so wonderful" (past tense - no longer)

## Competitive Context
- Descript is dominant in podcast editing (especially for text-based editing)
- No alternative mentioned - users stuck with it
- Problem is UPDATE CADENCE and BREAKING CHANGES, not core functionality

## Buyer Persona
- Podcasters (indie creators, likely solo or small teams)
- Already paying for Descript (subscription product)
- Time-constrained (extra editing time is painful)
- Want stability over new features

## Root Cause
SaaS product prioritizing rapid feature shipping over backward compatibility and user communication. Classic product management failure: optimizing for "shipping" metrics rather than user experience continuity.

## Market Gap
Podcast editing tool with:
- Stable interface
- Backward compatibility guarantees
- Clear release notes and migration guides
- LTS (Long Term Support) version option
- User control over when to update
